## language

- write everything in English: code, variable names, function names, comments, test descriptions, test file content, skill files, command files
- the only exception is commit messages, which follow the project's defined language

## geral
- sempre delete arquivos que não estão sendo usados
- sempre pergunte antes de commitar
- nunca faça commit em arquivos que ainda não estiverem prontos
- nunca faça nada que não foi solicitado
- ao fazer uma alteração de regra de negócios, atualize o arquivo relacionado à mudança em `docs/`
